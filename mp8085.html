Addressing Modes

For executing any instruction micro processor requires data. Hence, alongwith the instruction we have to give address of source data. The method by which the address of source of data is given in an instruction is called as addressing modes of source or simply addressing mode. The microprocessor will process the data and will produce reult. To store this result we will again give an address of destination of reult. This is called addressing mode of destination.


There are five types of addressing modes :

1. IMMEDIATE ADDRESSING MODE

If the 8 or 16 bit data required for executing the instruction is given along with the instruction , then such an instruction is called IMMEDIATE ADDRESSING MODE instruction. The instructions generally have last alphabet 'I'.
Ex: MVI A - move immediate data into accumulator
    LXI B -

2. DIRECT ADDRESSING MODE

If the 8 or 16 bit data required for executing the instruction is stored at some memory location and 16 bit address of this memory location is giver alongwith the instruction , then such instructions are called DIRECT ADDRESSING MODE instructions.
Ex: LDA 3000H - with this instruction , 8 bit data at mem location 3000H is transferred to accumulator directly.

3. REGISTER DIRECT ADDRESSING MODE

If the 8 or 16 bit data required for executing the instruction is stored in register/register pair and the name of the register pair is given along with the instruction, then it is called REGISTER DIRECT ADDRESSING MODE instructions.
Ex: MOV A,B - the data is moved from one register to another.

4. REGISTER INDIRECT ADDRESSING MODE

If the 8 or 16 bit data required for executing the instruction is stored in some memory location whose address is stored in register/register pair and the name of the regiter/register pair is given alongwith the instruction, it is called REGISTER INDIRECT ADDRESSING MODE instructions.
Ex: LDAX B - accumulator is loaded with the 8 bit data from the memory location whose address is stored in BC pair, 
  so it has indirect addressing mode.

5. IMPLICIT ADDRESSING MODE

When the address of source data as well as dstination of result is fixed, then there is no need to giv any operand along with the instruction , such instruction are called IMPLICIT ADDRESSING MODE.
Ex: CMA - Compliment accumulator. Here, the data is picked from accumulator, complimented and saved back to accumulator.

OPCODE and OPERAND

Microprocessor instruction is used to tell the microprocessor, to do a specified task. Each instruction contains two parts :

OPERATION CODE (OPCODE) : The first part of the instruction specifies the task to be performed.
OPERAND : The second part of the instruction is the data to be operated on .

TYPES OF 8085 INSTRUCTION

1.ONE/SINGLE BYTE INSTRUCTION (SBI)

Any instruction in microprocessor, in which number is not given as operand in the instruction , can be completely expressed by one byte of code or two digit hexacode. Such instruction is called SBI. 
Ex :  MOV A, B
      ADD C
      LDAX B

2.TWO/DOUBLE BYTE INSTRUCTION (DBI)

If 8 bit number is given in Operand, then to express such instructions completely , we need two 8 bit codes. Hence such instructions are called double byte instructions. 
Ex :  MVI A, 36H
      CPI   24H
       /      \
 First Bit     Second Bit

3.THREE/TRIPLE BYTE INSTRUCTION (TBI)

If 16 bit number is given in operand, then to express such instructions completely, we need three 8 bit codes. Hence such instruction are called TBI.
-First byte will be the opcode
-Second byte will be the 8 LSB's of 16 bit number
-Third byte will be the 8 MSB's of 16 bit number
Ex :  LXI H, 1536H
